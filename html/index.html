Rest backend usermanual



Objects:

Project:
- Required fields: name
- Optional fields: N/A

Issue:
- Required fields: summary
- Optional fields: text, closed(true/false)

Comment:
- Required fields: text
- Optional fields: N/A

Urls:

get all projects:
	/1/projects
	
get one project:
	/1/projects/<project-key>
	
get all project issues:
	/1/projects/<project-key>/issues
	
get one project issue
	/1/projects/<project-key>/issues/<issue-key>
	
get all issue comments:
	/1/projects/<project-key>/issues/<issue-key>/comments
	
get one issue comment:
	/1/projects/<project-key>/issues/<issue-key>/comments/<comment-key>

Methods:

Use rest to manipulate the objects:
- put	 (create)
- post	 (update)
- get	 (get)
- delete (delete)
